Chrome 通知 API 常用功能（能做什么）
只保留实际开发会用到的核心操作，废弃 / 无用功能全部剔除，一目了然：
一、创建各类通知
发送基础系统通知：标题 + 内容 + 图标，弹出在系统托盘
发送列表式通知：一条通知里展示多条条目信息
发送进度条通知：显示下载 / 上传 / 处理进度（0-100）
发送带按钮的通知：最多加 2 个操作按钮，用户点击可触发逻辑
发送静音通知：不播放提示音、不震动
发送常驻通知：不会自动消失，必须用户点击或手动关闭才会消失
二、管理已有通知
主动清除指定通知（关掉某条弹出的通知）
更新已有通知：修改内容、进度、标题等，不用重新创建
获取当前所有正在显示的通知列表
三、权限相关
查询用户是否允许当前扩展发通知（授权 / 拒绝）
四、监听用户交互
监听用户点击通知本身（非按钮区域）
监听用户点击通知上的按钮
监听通知被关闭（区分是用户手动关，还是系统自动关掉）
